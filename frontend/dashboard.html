🛠 CORS Notes for Browser Access

      

        REST API (8080): CORS fully enabled (allow_origins=["*"]). Works natively. 

        Triton (8000/8002), Redpanda (9644), Prometheus (9090), MinIO (9000): May not include CORS headers by default.
        
    
          
  • Use the Allow CORS browser extension during development.
    • 
          
  • Or open Chrome with --disable-web-security --user-data-dir=/tmp/cors-test
    • 
          
  • Or run this dashboard via a reverse proxy (nginx) that adds CORS headers.
    • 
          
  • MinIO — CORS can be configured in Console → Settings.
    • 
          
  • Prometheus — restart with --web.enable-lifecycle or use Grafana as proxy.
    • 
        

      

    

  








  

    
⚠ CORS Policy Blocked Some Requests

    

      Some services (Triton, Redpanda, Prometheus, MinIO) don't include CORS headers by default,
      so your browser blocks the requests.


      Quick Dev Fix (Chrome):
      
chrome.exe --disable-web-security --user-data-dir="C:\Temp\chrome-cors"

      Install extension:

      Allow CORS: Access-Control-Allow-Origin


      Permanent fix — serve via nginx with CORS headers.


      The REST API on port 8080 is fully CORS-enabled and will always work.
